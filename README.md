 **AfuBlog Contributor Guidelines**

Welcome! 👋 Thank you for contributing to **AfuBlog**, part of the AfuChat ecosystem.  
This guide explains how to add content while keeping the site consistent, professional, and minimal.

 **1️⃣ Design & UI Guidelines**

Keep it flat: No shadows, gradients, or card-like components.  
No rich text styling: Avoid bold, italics, underlines, or colored text in posts.  
Minimal content blocks: Each post should be clean and simple.  
Professional tone: Content should feel corporate, reliable, and informative.

Brand Colors:  
Primary: `#007BFF` (main accent color for buttons, links, and highlights — blue)  
Secondary / Neutral: `#FFFFFF` (background), `#000000` (text)  
Avoid introducing new colors. Always keep UI minimal and flat.

 **2️⃣ Content Guidelines**

Only text and plain images.  
Avoid multimedia such as videos or GIFs inside posts.  
Do not include ads or promotional links.  
Ensure content is factual, concise, and company-style.

**3️⃣ Workflow**

Create a branch from `dev` or your assigned branch.  
Add or update content according to the guidelines above.  
Submit a Pull Request (PR) to `main` after finishing edits.  
Wait for review: All PRs require at least 1 approval before merging.

**4️⃣ Technical Notes**

Project uses TypeScript + React.  
Keep component structure simple — follow existing patterns.  
Avoid installing unnecessary packages or styling libraries.  
Commit messages should be clear, e.g., `Update blog post about AfuChat`.

## 5️⃣ Contact / Questions

If unsure about style or workflow:

Comment on the Pull Request  
Refer to existing posts for consistency

Remember: AfuBlog should always feel professional and company-like.  
Keep it simple, flat, minimal, and always consistent with the brand color `#007BFF`.
